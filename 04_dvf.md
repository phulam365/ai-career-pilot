# DVF Framework: AI Career Path Copilot

## Desirable (for users)
**Yes.** Early-career tech learners in Vietnam face high uncertainty when translating university coursework, certificates, GitHub projects, and interests into real job options. AI Career Path Copilot directly answers the user's most urgent questions: which role am I closest to, what am I missing, what should I learn next, what project should I build, and which jobs can I apply for now?

The product is especially desirable because it turns generic career advice into personalized, explainable guidance. Users do not just receive a recommendation; they see the evidence behind the match, such as skills found in their CV, GitHub signals, missing requirements, and alignment with selected target roles.

## Viable (for businesses)
**Yes.** The product can create value for multiple stakeholders:
*   **Students and junior workers:** Freemium access to basic assessment, with paid upgrades for deeper CV review, portfolio guidance, or interview preparation.
*   **Universities and training providers:** Cohort dashboards that help career centers identify common skill gaps and improve employability outcomes.
*   **Recruiters and employers:** Better prepared junior candidates whose skills and portfolios are aligned with target roles.

The strongest initial business path is B2C freemium for hackathon demonstration, followed by B2B partnerships with universities, bootcamps, or career centers after validation.

## Feasible (for engineers)
**Yes for an MVP prototype.** The hackathon version can be built as a frontend-first prototype with mocked or lightweight AI analysis:
*   **Inputs:** CV upload, optional GitHub link, target role selection, and basic self-assessment.
*   **Processing:** LLM-assisted extraction of skills and projects from CV text, rule-based or prompt-based matching against role requirements, and explainable gap analysis.
*   **Outputs:** Role match scores, skill gaps, 30-day learning plan, portfolio project recommendation, and sample internship or junior job matches.

The MVP should avoid over-promising real hiring outcomes. It should focus on explainability, practical next steps, and user trust. Production versions would require stronger data privacy controls, job market data integrations, robust GitHub parsing, and validation from career advisors or recruiters.
